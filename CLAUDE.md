## Project Overview

Allia Blog is a Hugo-based static blog that serves as an automated content distribution hub for Allia Studio. Posts are generated by an AI content marketing system (Research-Project) and automatically committed via GitHub API. The blog publishes to GitHub Pages and provides an RSS feed consumed by the main landing page (allia.dev).

## Build and Development Commands

```bash
# Start local development server (includes drafts)
hugo server -D

# Build for production
hugo --minify

# Create a new post using the archetype template
hugo new posts/my-post-slug.md
```

The site builds automatically via GitHub Actions on push to `main`. Hugo version 0.124.0 (extended) is used in CI.

## Architecture

```
Research-Project (AI agents)
    ↓ commits posts via GitHub API
content/posts/*.md
    ↓ Hugo build (GitHub Actions)
public/ → GitHub Pages
    ↓ RSS feed
allia-landingpage (Astro) consumes at allia.dev
```

**Key configuration:** `hugo.toml` - language is Spanish (es), theme is `allia-theme`, outputs HTML + RSS.

## Theme Structure

Custom theme at `themes/allia-theme/`:
- `layouts/_default/baseof.html` - Base template with header/footer
- `layouts/_default/single.html` - Individual post layout
- `layouts/_default/list.html` - Posts list with cards
- `layouts/index.html` - Home page (hero + 10 recent posts)
- `static/css/style.css` - Complete responsive stylesheet

CSS uses variables: `--color-primary: #2563eb`, `--max-width: 720px`, system fonts.

## Post Format

Posts in `content/posts/` use this frontmatter structure:
```yaml
---
title: "Article Title"
slug: "article-slug"
date: 2025-01-23T12:00:00-05:00
draft: false
author: "Allia Studio"
description: "Meta description"
categories: ["Tech", "AI"]
tags: ["tag1", "tag2"]
---
```
